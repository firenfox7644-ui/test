Section I - Objective                                                                                                                                                                                                                                                      
  • The platform should work well alongside our existing marketing and analytics setup — we don't want data silos or redundant functionality.                                                                                                                               
  • App performance is non-negotiable — no slowdowns, no battery drain, no bloated app size                                                  
  • Robust offline handling given the nature of airline travel (airplane mode, poor connectivity zones)


Functional Detailed Requirements  :                                                                                                                                                                                                                         
Offline & Connectivity Handling                                                                                                        
  • The platform should handle offline scenarios gracefully without breaking app functionality                                               
  • Campaign content should be cached locally so users see relevant content even without connectivity                                        
  • Cache storage should be managed sensibly to avoid excessive device storage usage                                                         
  • User events occurring during offline periods should be queued and synced automatically when connectivity returns                         
  • Queued events should be preserved even if the app is terminated while offline                                                            
  • The platform should handle airplane mode seamlessly — a common scenario for airline passengers                                           
  • Transitions between online and offline states should be smooth with no error messages shown to users  

Non-Functional Detailed Requirements:
  Add SDK performance section:                                                                                                                                                                                                                                                     
  SDK Performance:                                                                                                                                                                                                                                                                   
  • The SDK should initialise quickly and should not block app launch or delay the main thread                                               
  • SDK binary size should be reasonable and not significantly increase app download size                                                    
  • Runtime memory usage should be optimised and should not impact app performance                                                           
  • The SDK should not cause noticeable impact on app cold start time                                                                        
  • Campaign rendering should feel instantaneous to users                                                                                    
  • Background SDK operations should have minimal impact on battery consumption                                                                                                                                                                                                          
  Vendor should provide benchmark data from comparable production deployments. IndiGo will validate performance during the POC phase.   
 
  Emergency Controls & Failsafe Mechanisms:                                                                                                                                                                                                                                            
  • The platform should provide a remote kill switch to disable the SDK without requiring an app update                                      
  • Individual campaigns should be stoppable immediately if issues are detected                                                              
  • The SDK should implement sensible retry logic with backoff to avoid excessive network calls during failures                              
  • Network operations should have configurable timeouts                                                                                     
  • SDK errors should be isolated and should never cause the host app to crash                                                               
  • The platform should offer rollback capability to revert to previous configurations if needed                                             
  • If the vendor backend is unreachable, the SDK should fall back to cached content or remain silent — users should not see error states
 
  Add new section — Campaign Analytics:                                                                                                                                                                                                                                                   
  • The platform should provide real-time visibility into campaign metrics including impressions, clicks, conversions, and dismissals        
  • Performance breakdowns by user segment or cohort should be available                                                                     
  • Multi-step journeys should have funnel visualisation showing progression and drop-off                                                    
  • The platform should support revenue or conversion attribution where campaigns are tied to business outcomes                              
  • Reports should be exportable in common formats and available on-demand                                                                   
  • Campaign analytics data should be accessible via API for integration with external systems
